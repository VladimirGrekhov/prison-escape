# 🔒 Prison Escape

Браузерная настольная игра «бросай кубики» на 4 игрока с онлайн-мультиплеером
в реальном времени. Доска рисуется на `<canvas>`, поддержаны светлая/тёмная
тема и игра как на одном устройстве (hot-seat), так и онлайн через
авторитативный сервер на [Colyseus](https://colyseus.io/).

🎲 Играть: <https://prison-escape.duckdns.org/>

## Как это устроено

Чистый статический фронтенд (HTML / CSS / canvas JS) + отдельный Node-сервер
для мультиплеера. Сервер **авторитативный**: он владеет кубиками, порядком
ходов и распределением мест, поэтому клиент не может бросить вне очереди или
подделать результат. Если сервер недоступен — игра полностью работает офлайн
в режиме hot-seat на одном устройстве.

### Структура

```
index.html        точка входа
css/style.css     стили, темы, адаптивная вёрстка
js/
  board.js        отрисовка доски на canvas (геометрия задаётся константами)
  game.js         игровая логика, анимация кубиков, переключение тем
  net.js          клиент Colyseus: выбор endpoint, синхронизация состояния
  sound.js        звуки кубиков
  colyseus.js     бандл colyseus.js@0.16.22 (должен совпадать с версией сервера)
server/
  index.js        авторитативный сервер: комната `prison`, 4 места + зрители
  package.json    colyseus@0.16 + @colyseus/schema@3
```

## Запуск локально

Фронтенд — статика, достаточно любого http-сервера из корня репозитория:

```bash
python3 -m http.server 8000
# открыть http://localhost:8000
```

Сервер мультиплеера (опционально — без него работает офлайн hot-seat):

```bash
cd server
npm install
node index.js        # слушает ws://127.0.0.1:2567
```

Клиент сам выбирает endpoint в `js/net.js`: на `localhost`/`file://` он
подключается к `ws://localhost:2567`, в продакшене — к `wss://<host>/mp`.

> ⚠️ Версии Colyseus на клиенте и сервере должны совпадать (0.16 / schema 3).
> Не обновляйте сервер до colyseus 0.17 / schema 4 — формат сообщений
> разойдётся с бандлом `js/colyseus.js`.

## Геймплей

- До 4 игроков занимают углы доски; лишние подключения становятся зрителями.
- Ход по очереди: активный игрок бросает два кубика, ход автоматически
  переходит дальше.
- Имя задаётся в поле ввода, кнопка «↺ Сброс» перезапускает партию.
- Отключившемуся игроку даётся 30 секунд на переподключение, прежде чем
  его место освободится.

## Деплой

Прод обслуживается nginx (статика) + systemd-юнит для сервера мультиплеера за
обратным прокси на `/mp/` (WebSocket upgrade), TLS через Let's Encrypt.
